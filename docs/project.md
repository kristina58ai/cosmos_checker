# Cosmos Checker — Project Brief

## Название
Cosmos Checker

## Описание
Десктоп-приложение (Windows) с графическим интерфейсом (GUI) для массовой проверки криптовалютных кошельков в экосистеме Cosmos SDK. Аналог Mintscan по покрытию сетей, но с фокусом на batch-проверку: пользователь загружает txt-файлы с кошельками (адреса, seed-фразы или приватные ключи), назначает прокси, и получает отчёт по балансам и контрактам (стейкинг, delegation rewards, unbonding и т.п.) по всем выбранным сетям.

## Цели
1. Максимальное покрытие Cosmos-сетей (все сети из Chain Registry / Mintscan)
2. Высокая скорость массовой проверки: минимум 5000 кошельков/минуту
3. Проверка балансов и контрактов где хранятся средства: native-балансы, стейкинг (delegations), delegation rewards, unbonding, IBC-токены
4. Удобный импорт кошельков и экспорт результатов

## Целевые пользователи
- Операторы нод и валидаторов
- Инвесторы и холдеры крипто-активов
- Разработчики на Cosmos SDK
- Широкая аудитория с разным уровнем технической подготовки

## KPI
1. Количество поддерживаемых Cosmos-сетей (цель: все из Chain Registry)
2. Скорость проверки кошельков (цель: ≥5000 кошельков/мин)
3. Стабильность работы на Windows

## Definition of Done
Приложение с GUI запускается на Windows. Пользователь видит все сети Cosmos (как на Mintscan). Выбирает нужные сети → загружает txt-файлы с кошельками (адреса, seed-фразы или приватные ключи) → загружает txt-файл с прокси → запускает проверку → получает данные о балансах и контрактах кошельков. Минимальная скорость: 5000 кошельков/мин. Результаты экспортируются в txt.

## Скоуп — ЧТО ВХОДИТ
- Графический интерфейс (GUI) для Windows
- Автодискаверинг всех Cosmos-сетей из Chain Registry
- Выбор сетей для проверки
- Импорт кошельков из txt-файлов: адреса, seed-фразы, приватные ключи
- Деривация адресов из seed-фраз и приватных ключей (см. раздел ниже)
- Импорт и ротация прокси из txt-файла
- Параллельная проверка кошельков по выбранным сетям
- Получение данных: native-балансы, IBC-токены, стейкинг (delegations), delegation rewards, unbonding delegations
- Экспорт результатов в txt
- Поддержка нескольких кошельков одновременно

## Деривация адресов из seed-фраз и приватных ключей

### Из seed-фразы (мнемоника BIP39):
1. Seed-фраза (12/24 слова) → BIP39 seed (512 бит) через PBKDF2-HMAC-SHA512
2. BIP39 seed → мастер-ключ через HMAC-SHA512 (стандарт BIP32)
3. Мастер-ключ → деривация по HD-пути Cosmos: `m/44'/118'/0'/0/0` (coin_type 118 для Cosmos Hub; у других сетей свой coin_type, берётся из Chain Registry: поле `slip44`)
4. Дочерний приватный ключ → публичный ключ (secp256k1, сжатый 33 байта)
5. Публичный ключ → SHA256 → RIPEMD160 → 20 байт (address bytes)
6. Address bytes → Bech32 кодирование с HRP (human-readable prefix) конкретной сети (например: `cosmos`, `osmo`, `juno` — берётся из Chain Registry, поле `bech32_prefix`)

### Из приватного ключа (hex):
1. Приватный ключ (32 байта) → публичный ключ (secp256k1, сжатый 33 байта)
2. Далее шаги 5-6 из процесса выше

### Важно:
- Один seed может дать адреса для ВСЕХ выбранных Cosmos-сетей (разные bech32-префиксы, но возможно разные coin_type/HD-пути)
- Для каждой сети из Chain Registry берём: `slip44` (coin_type), `bech32_prefix`, и при необходимости кастомный HD-путь
- Приватные ключи и seed-фразы хранятся ТОЛЬКО в оперативной памяти на время работы, НЕ сохраняются на диск
