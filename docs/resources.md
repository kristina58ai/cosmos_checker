# Cosmos Checker — Resources & Constraints

## Время
- Дедлайн: нет (личный проект)
- Разработка ведётся solo-разработчиком

## Бюджет
- Минимальный: бесплатные API и сервисы
- Допускается использование платных API при необходимости

## Целевая платформа
- **OS:** Windows (обязательно), кросс-платформенность — бонус
- **Тип:** Десктоп-приложение

## Тех-стек (предварительный)
- Выбор на усмотрение архитектора
- Приоритет: максимальная скорость обработки кошельков
- Обязательно: стабильная работа на Windows

## Внешние API и источники данных

### Основные (комбо-стратегия с автопереключением)

| Приоритет | Источник | Протокол | Назначение |
|-----------|----------|----------|------------|
| 1 | **gRPC эндпоинты** | gRPC (Protobuf) | Основной протокол — максимальная скорость |
| 2 | **LCD/REST API** | HTTP REST | Фоллбэк, если gRPC недоступен |
| 3 | **cosmos.directory** | REST proxy | Резервный с load balancing и health-check |

### Вспомогательные

| Источник | Назначение |
|----------|------------|
| **Cosmos Chain Registry** | Дискаверинг сетей, эндпоинтов, метаданных (bech32, chain_id, токены) |
| **CosmJS** (если JS-стек) | Клиентская библиотека для взаимодействия с Cosmos |

## Ограничения
- Приложение полностью **read-only** — никаких записывающих операций в блокчейн
- Seed-фразы и приватные ключи используются ТОЛЬКО для деривации адресов, НЕ для подписи транзакций
- Прокси необходимы для обхода rate limits публичных нод
- Нужен graceful fallback: если gRPC не отвечает → LCD → cosmos.directory
- Минимальная скорость: 5000 кошельков/мин при batch-проверке
- Входные данные: txt-файлы (кошельки, прокси)
- Выходные данные: экспорт в txt
- Обязателен графический интерфейс (GUI)
